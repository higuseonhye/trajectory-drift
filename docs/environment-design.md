# Environment & state design

Organizations and individuals drift in **dead environments** — not only through bad decisions.

Canonical personal-layer doc: [trajectory-native/docs/environment-design.md](https://github.com/higuseonhye/return/blob/main/docs/environment-design.md)

---

## At org / system scale

| Dead environment | Drift effect |
|------------------|--------------|
| Fluorescent open-plan execution theater | Numb coordination, mission fade |
| Always-on async / scroll culture | Signal loss, no awe or depth |
| Sterile "productivity" tooling | Metrics fine, humans dead inside |
| Fake networking / performative standups | Social energy without truth |
| Extractive velocity culture | Functional collapse |

Teams can look **operational** while environment drains vitality and direction.

---

## Environmental drift detection

Signals to watch:

- prolonged indoor/sterile work patterns in event streams
- interaction starvation + high async volume
- coordination without meaningful contact
- institutional memory in lifeless systems nobody chose

Feeds Drift Radar alongside structural drift taxonomies.

---

## Recovery (environment-aware)

Not only process fixes — **atmospheric and state restoration**:

- spaces and rituals that restore aliveness
- meaningful synchronous contact
- sensory and aesthetic coherence in how teams work
- anti-dead-environment design in tooling and culture

See [`../framework/drift-taxonomy/`](../framework/drift-taxonomy/) · [native environment framework](https://github.com/higuseonhye/return/tree/main/framework/environment-design)
