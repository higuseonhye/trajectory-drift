# Behavioral patterns

Recurring behavioral loops that destroy trajectory coherence.

Each pattern is a **detectable loop** — not a character flaw. Product infrastructure should surface and interrupt these loops.

---

## Status optimization loop

```
seek visibility → get validation → optimize for more visibility → reduce asset building
```

**Drift type:** Prestige Drift

**Signals:** Posting > building; credential accumulation; networking without ownership outcomes.

**Intervention:** Force asset milestone; reduce visibility inputs; track creation ratio.

---

## Pseudo-productivity loop

```
feel busy → optimize workflows → add tools → feel productive → no assets created
```

**Drift type:** Labor Drift · AI Drift

**Signals:** Tool accumulation; calendar density; task completion metrics; no leverage.

**Intervention:** Ownership classification on events; compounding analysis.

---

## Reactive internet loop

```
check feed → react → lose focus → feel behind → check feed for solutions
```

**Drift type:** Noise Drift

**Signals:** Context switching; trend-chasing; urgency-driven decisions.

**Intervention:** Noise reduction; batch communication; protect deep work.

---

## Low-agency cycle

```
encounter obstacle → consume content about obstacle → feel informed → avoid action
```

**Drift type:** Apprentice Drift

**Signals:** Learning hours >> execution hours; courses without projects.

**Intervention:** Application milestones; ship deadlines; close learning loops.

---

## Prestige-seeking pattern

```
see successful person → imitate surface → skip compounding work → repeat with next person
```

**Drift type:** Prestige Drift

**Signals:** Strategy switching; copying aesthetics not systems; no enduring direction.

**Intervention:** Trajectory graph continuity check; long-term project commitment.

---

## Fragmentation spiral

```
new idea → start project → lose momentum → new idea → repeat
```

**Drift type:** Multiple — Noise, Apprentice, Organizational

**Signals:** Many open projects; no closed loops; no leverage accumulation.

**Intervention:** WIP limits; loop closure metrics; kill or commit decisions.

---
